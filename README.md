## C4 model
It is a lightweight approach to describing your software architecture
- Context diagram - Shows the system in its environment: users, external systems, and high-level interactions
- Container diagram - shows the objects running inside a system
- Component diagram - shows the building blocks that make up each container
- Code diagram - rarely used, typically UML or ER diagram
## Diagram Types [packtpub Link](https://www.packtpub.com/en-us/product/tools-and-skills-for-net-10-9781835882504)

| Type               | Description |
|--------------------|-------------|
| flowchart, graph   | Flowcharts and graphs allow you to visualize steps and decision points in a process or workflow, making complex processes easier to understand at a glance. |
| sequenceDiagram    | Used to show how objects interact in a given sequence, ideal for representing the flow of messages, events, or actions between different parts of a system. |
| classDiagram       | Depicts the structure of a system by showing its classes, attributes, operations, and relationships among objects. |
| stateDiagram       | Illustrates the states an entity can be in and the transitions between those states. Useful for modeling system or application behavior. |
| erDiagram          | Used in database design and modeling to show relationships between data or entities in a database. |
| gantt              | Used in project management to provide a visual timeline of tasks, dependencies, and durations throughout a project lifecycle. |
| journey            | Shows a user journey when interacting with a system. |
| gitGraph           | Visually displays branches in a Git audit history. |
| pie                | Represents data distributions or proportions in a simple and clear way. |
| mindmap            | Represents relationships between ideas during brainstorming. |
